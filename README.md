 1 - Etapa Conceitual

O que é controle de versões e por que ele é importante:
O controle de versões é um sistema que registra todas as alterações feitas em um projeto ao longo do tempo. Ele permite voltar a versões anteriores, comparar mudanças, identificar quem fez cada modificação e trabalhar em equipe sem perder dados. É importante porque evita perda de arquivos, facilita o trabalho colaborativo e organiza o histórico de desenvolvimento do software.
 
Diferença entre Git e GitHub:
O Git é uma ferramenta local de controle de versões, usada diretamente no computador. Já o GitHub é uma plataforma online onde os repositórios criados com o Git podem ser armazenados e compartilhados.
Resumindo: Git é a ferramenta, e GitHub é o site que hospeda os projetos criados com ela.

Principais vantagens de utilizar controle de versões:
Histórico completo de mudanças.
Possibilidade de reverter erros.
Trabalho colaborativo sem conflito de arquivos.
Integração com plataformas como GitHub ou GitLab.
Segurança e backup do projeto.


Comandos básicos do Git e o que fazem:
git init → Inicia o controle de versões em uma pasta.
git status → Mostra o estado atual do repositório (quais arquivos foram alterados ou adicionados).
git add . → Adiciona arquivos ao controle (prepara os arquivos para serem salvos).
git commit -m "mensagem" → Salva uma nova versão do projeto com uma mensagem de identificação.
git log → Mostra o histórico de todas as versões salvas.
git remote add origin <url> → Conecta o projeto local a um repositório no GitHub.
git push -u origin main → Envia os arquivos locais para o repositório online.
git pull → Atualiza o projeto local com a versão mais recente do GitHub.
